# Full Implementation Plan — GutGutGoose Take-Home

> Philosophy: Build the minimum thing that tells the most honest story. Every file has a reason. No code without understanding why it exists.

## The System in One Sentence

A Python FastAPI backend processes a synthetic metagenomic sample through ML scoring, then calls Claude to generate a personalized plain-English report, which a React frontend displays as a beautiful, science-backed gut health profile for Susan.

## Tech Stack Decisions

| Layer | Choice | Why NOT the alternative |
|-------|--------|------------------------|
| Backend | FastAPI | Flask lacks async; async needed for Claude API calls. Auto-generates /docs. Pydantic v2 validation built-in. |
| Frontend | React + Vite | Next.js adds SSR complexity not needed for a demo. Vite cold starts in 300ms. |
| Charts | Chart.js | D3.js is too low-level for a prototype. Recharts is React-only (Chart.js has wrappers). |
| Styling | Tailwind CSS | Faster than writing CSS from scratch. GutGutGoose uses a clean minimal aesthetic. |
| LLM | Claude API (claude-sonnet-4-5) | We're demoing for an AI company. Shows alignment. claude-sonnet-4-5 has structured output. |
| ML | scikit-learn + numpy | PyTorch is overkill. The models here (KMeans, scoring functions) don't need GPU. |
| Explainability | SHAP library | Standard in industry ML. Shows we know how to explain model decisions. |
| Data | JSON + TSV flat files | SQLite adds setup friction. For a demo, files are fine. |
| Reference data | Hardcoded HMP stats | curatedMetagenomicData requires R. Pre-computing the percentile tables means zero runtime dependency. |

**Purpose**: Creates a realistic (but small) paired-end FASTQ that represents Susan's gut community.

**Approach**: For each species in Susan's profile, we sample random DNA sequences at the correct relative frequency. In real life these would be real genome fragments — for the prototype, we generate plausible GC-content-matched sequences.

```python
"""
```

## Phase 2 — ML Layer

**Time: 4–5 hours**
**Deliverable**: All files in `ml/`, producing a structured `ml_results` dict

### `ml/clr_transform.py`

```python
"""
THE MOST IMPORTANT PREPROCESSING STEP.
Must be applied before any distance calculation, clustering, or regression.

Why: Microbiome abundances are compositional (sum to 100%).
     This creates spurious negative correlations and breaks standard statistics.
     CLR (Centered Log-Ratio) removes this constraint.

clr_transform(df: pd.DataFrame) -> pd.DataFrame
    Input: species × samples matrix, values are raw relative abundances
    Output: same shape, CLR-transformed (unconstrained)
    Pseudocount 0.5 added before log to handle zeros (standard practice)
    
    Formula: CLR(x_i) = log(x_i / geometric_mean(x))
    
    After this transform: Euclidean distance = Aitchison distance
    (the gold standard distance metric for compositional data)
"""
```

### `ml/diversity.py`

```python
"""
shannon_index(abundances: dict) -> float
    H = -Σ(p_i * log(p_i))  where p_i = abundance / 100
    0 = single species dominates, ~3-4 = healthy diverse gut
    Susan's expected: ~2.8 (moderately diverse but below optimal)
    
    Clinical note: Shannon index below 2.0 is associated with dysbiosis.
    Typical healthy adult: 2.5–3.5.

simpson_index(abundances: dict) -> float
    D = 1 - Σ(p_i²)
    0–1 range, higher = more diverse
    Less sensitive to rare species than Shannon

observed_species(abundances: dict, threshold=0.01) -> int
    Count of species above 0.01% — species richness
"""
```

### `ml/gut_score.py`

```python
"""
The health score and its explanation. This is the number on Susan's report.

GutScoreResult:
    score: float          # 0–100
    grade: str            # "Excellent" / "Good" / "Below Average" / "Dysbiotic"
    contributions: list   # Per-species contribution to score (SHAP-style)
    
score_components:
    1. Species balance score (60% of total):
       Each key species contributes based on deviation from reference mean.
       Weights are derived from clinical significance in the literature.
       
       BENEFICIAL (contributes positive when above mean, negative when below):
         F. prausnitzii    weight: +3.5  (most important butyrate producer)
         Akkermansia       weight: +3.0  (mucus barrier, metabolic health)
         Bifidobacterium   weight: +1.5  (immune regulation, B-vitamins)
         Roseburia         weight: +1.5  (butyrate producer)
         
       FLAGGED (contributes negative when above threshold):
         E. coli > 1%      weight: -2.5  per 1% above threshold
         Ruminococcus gnavus > 3%  weight: -1.5  (IBD-associated)
       
    2. Diversity score (25% of total):
       Shannon index normalized to 0–25 range
       Shannon 3.5+ → full 25 points
       Shannon < 2.0 → 0 points
       
    3. Firmicutes/Bacteroidetes ratio (15% of total):
       Ratio 0.5–2.0: full 15 points (healthy range)
       Outside range: scaled deduction

calculate_gut_score(profile: AbundanceProfile, user: UserProfile, 
                    reference: ReferenceData) -> GutScoreResult

Susan's expected score: ~47/100
Breakdown: Base 50 | F.prausnitzii deficit -8 | Akkermansia deficit -10 | 
           E.coli excess -6 | Diversity +8 | F/B ratio +13 = 47
"""
```

```python
      E. coli: 97th percentile (higher than 97% — alarming direction)
```

### `ml/enterotyping.py`

```python
"""
Classifies Susan into one of 3 gut "types" (enterotypes).
Reference: Arumugam M, et al. (2011) Nature 473:174-180.

Enterotypes:
    ET1: Bacteroides-dominated  → Western diet, protein-rich
    ET2: Prevotella-dominated   → Plant-based, fiber-rich  
    ET3: Ruminococcus-dominated → Mixed; often higher diversity, common in healthy elderly

Implementation:
    1. Collapse species → genus level (sum abundances within genus)
    2. CLR-transform genus-level data
    3. Use pre-fitted k-means centroids (k=3) from published study
       (We hardcode the centroids rather than retraining — they're published values)
    4. Assign Susan to nearest centroid
    5. Report: enterotype name + distance to centroid (how "pure" the assignment is)

susan_enterotype() -> EnterotypingResult:
    type: "ET1"  # Bacteroides-dominant (22% B. vulgatus is strongest signal)
    secondary: "ET2"  # Prevotella presence (13%) makes this a mixed profile
    purity: 0.61  # 0-1, lower = more mixed type
    interpretation: "Mixed Bacteroides/Prevotella profile — common in..."
"""
```

### `interpretation/prompts.py`

```python
"""
All Claude prompts live here. This keeps interpretation/interpret.py clean.
One prompt template per finding type.

FINDING_PROMPT: str — for each key species finding
    Variables: {species}, {abundance}, {percentile}, {reference_median},
               {primary_function}, {clinical_associations}, {user_goals}
    
    Constraints in prompt:
    - Max 80 words (report card, not essay)
    - No medical advice language ("you should see a doctor about this")
    - No probiotics mention (GutGutGoose's preference)
    - MUST reference the user's stated goal (e.g., "energy", "bloating")
    - Plain language: the test is "would a 42-year-old non-scientist understand this?"

SEX_FINDING_PROMPT: str — for estrobolome/hormonal findings
```

```bash
#!/bin/bash
# REAL PIPELINE — runs on Linux/WSL only
# Documents exact commands to process a real FASTQ file

# Step 1: Quality control
fastqc data/raw/sample_R1.fastq data/raw/sample_R2.fastq -o data/processed/

# Step 2: Adapter trimming and quality filtering  
fastp \
  -i data/raw/sample_R1.fastq \
  -I data/raw/sample_R2.fastq \
  -o data/processed/trimmed_R1.fastq \
  -O data/processed/trimmed_R2.fastq \
  --detect_adapter_for_pe \
  --cut_right \
  --cut_window_size 4 \
  --cut_mean_quality 20 \
  --length_required 50 \
  --json data/processed/fastp_stats.json \
  --html data/processed/fastp_report.html \
  --thread 8

# Step 3: Remove human host reads (CRITICAL before classification)
# Download hg38 index: bowtie2-build hg38.fa hg38_index
bowtie2 -x hg38_index \
  -1 data/processed/trimmed_R1.fastq \
  -2 data/processed/trimmed_R2.fastq \
  --un-conc-gz data/processed/dehost_R%.fastq.gz \
  -S /dev/null \
  --very-sensitive \
  --threads 8

# Step 4: Taxonomic profiling with MetaPhlAn 4
# Database: ~3GB download — metaphlan --install
metaphlan \
  data/processed/dehost_R1.fastq.gz,data/processed/dehost_R2.fastq.gz \
  --bowtie2out data/processed/metaphlan_bowtie2.bz2 \
  --nproc 8 \
  --input_type fastq \
  -o data/processed/susan_metaphlan.tsv

# This TSV is what our Python pipeline reads.
# Everything upstream of this is bioinformatics infrastructure.
# Everything downstream is where we add value.
```
